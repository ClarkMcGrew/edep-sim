/edep/

EDepSim detector control.

Sub-directories

/edep/gdml/Control over the gdml file.
/edep/material/Extra control over the material definitions.
/edep/phys/Control the physics lists
/edep/random/Control of the random seed number.
/edep/db/Output file control commands.

update

Do the run manager initialization. This causes the geometry to be built. This MUST be applied before "/run/beamOn".

Available Geant4 state(s) : PreInit

printMass [volume-name]

Print the mass of the first physical volume containing the sub-string. (set before update)

Available Geant4 state(s) : PreInit

Parameters
volume-name type s

validateGeometry

Check the geometry for overlaps (set before update).

Available Geant4 state(s) : PreInit

export [RootFile]

Export the geometry to a file. This is not compatible with event generation, or reading of kinematics files.

Available at all Geant4 states.

Parameters
RootFile type s

control [Name] [Version]

Set the run conditions for this simulation. This takes the name of a run condition and a version. This is translated into a G4 macro file which is executed. The run condition macros are saved in the <package-root>/src directory (e.g. src/baseline-1.0.mac).

Available at all Geant4 states.

Parameters
Name type s
Version type s

hitSagitta [Sensitive] [Sagitta] [Unit]

Set the maximum sagitta for hit segments.

Available Geant4 state(s) : PreInit

Parameters
Sensitive type s
Sagitta type d
Unit type s

hitSeparation [Sensitive] [Separation] [Unit]

Set the maximum separation for hit segments.

Available Geant4 state(s) : PreInit

Parameters
Sensitive type s
Separation type d
Unit type s

hitLength [Sensitive] [Length] [Unit]

Set the length for hit segments to stop growing.

Available Geant4 state(s) : PreInit

Parameters
Sensitive type s
Length type d
Unit type s

hitExcluded [LogicalVolume]

Exclude logical volumes from being sensitive.

Available Geant4 state(s) : PreInit

Parameters
LogicalVolume type s

runId []

This is the first run id that will be used by GEANT. GEANT will automatically increment the run id every time it starts a new internal run so care must be taken if you depend on this to pass-through the run id.

Available at all Geant4 states.

Parameters
type i

subrunId []

Set the sub run context field for documentation purposes. This has no internal meaning.

Available at all Geant4 states.

Parameters
type i
